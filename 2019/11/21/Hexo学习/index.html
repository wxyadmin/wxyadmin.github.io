Hexo学习

Veröffentlicht am 2019-11-21

白茶清欢无别事，我在等风也等你
纸上得来终觉浅，觉知此事要躬行
自有搭建个人博客的想法的那一刻开始到现在应该也快一个星期了，我就把我这一星期搭建hexo博客的过程复现一遍

以下演示基于windows 10系统，所有以下在windows10 64位下进行演示。

  1. 安装 Node.js
    • 因为 hexo 是基于 node 框架的，所以呢，我们首先需要下载安装node.js。
      下载地址
    • 安装步骤一直自定义下一步，如需更改安装路径将C改成D即可，最后点击Finish。
  2. 安装Git
    • 下载地址
    • 选择适合自己系统的安装即可，同样，默认一直选择下一步安装即可。安装成功在开始菜单这个Git Bash下载下来就相当于Linux中的终端窗口了，以后我们就用这个东东来打开终端。
  3. 安装hexo
    • 新建一个文件夹，例如我建的blog
    • 打开文件夹，然后在空白处点鼠标的右键，Git Bash Here
    • 建好后进去右键 git bash here 输入 node -v 看会不会显示出版本号,接着输入npm -v同理看有无版本号
    • 接下来安装cnpm，在上面的终端中输入 cnpm install -g hexo-cli hexo init hexo s 这个插件相当于是沟通你本地数据与github仓库的通道,之后在每次本地数据编辑完后都要用到它
      首先查看目录C:Users\用户名
      有没有.ssh目录,如果没有，在本地blog文件内打开git bash hexo,输入命令: 然后按3个回车，再检查有没有.ssh目录
      打开id_rsa.pub这个文件，进入你刚才创建的仓库，设置-部署公钥-新建部署公钥，公钥名称可以不填，将刚才复制的公钥粘贴在公钥内容里面
  4. 修改站点配置文件 deploy 参数内容
    到文件夹根目录下找config.yml的配置文件进去
    type后加git 并添加repo: https://github.com/youname/yourname.git/ ; branch: master repo后面的网址就是新建仓库的网址
    接下来就是搭建的最后一步 : bit bash 中输入hexo d 进行最终的部署工作, 这时它会让你填你的username和password 作为进入仓库的凭证

注意一下，这一步输错的话呢，就的重头再来了
还要注意的呢就是冒号后面要有空格
